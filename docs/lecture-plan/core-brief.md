## 3. 학습 목표 (KSA Framework)

### Knowledge (지식) - "알게 된다"
1. AI 시각화 도구 4종(노트북LM, 제미나이, 클로드, 안티그래비티)의 **역할과 강점**을 설명할 수 있다
2. 딥리서치 → 데이터 분석 → 시각화 → 보고서의 **워크플로우 단계**를 이해한다
3. 효과적인 데이터 시각화의 **원칙**(차트 선택, 색상, 레이아웃)을 안다

### Skill (기술) - "할 수 있다"
1. 노트북LM으로 다량의 자료를 **정리하고 인사이트를 추출**할 수 있다
2. 제미나이 딥리서치로 **트렌드/경쟁사 분석 보고서**를 생성할 수 있다
3. 클로드로 **데이터 시각화 코드와 보고서 구조**를 자동 생성할 수 있다
4. 안티그래비티로 **프로페셔널 슬라이드/인포그래픽**을 제작할 수 있다
5. 4개 도구를 **조합한 E2E 워크플로우**를 실행할 수 있다

### Attitude (태도) - "느끼게 된다"
1. "AI 도구를 **조합**하면 전문가급 결과물이 가능하다" → **자신감**
2. "이제 시각화는 디자이너가 아니라 **기획자의 영역**이다" → **주체성**
3. "매번 새로 만들 필요 없이 **워크플로우를 재활용**하면 된다" → **효율성 사고**

### 성공 기준 3단계

| Level | 기준 | 구체적 지표 |
|-------|------|------------|
| **L1: 최소 달성** | 4개 도구에 각각 접속하여 기본 기능 1개씩 사용 | 수강생 100% |
| **L2: 기대 달성** | 주어진 주제로 AI 시각화 보고서 1편 완성 | 수강생 80% |
| **L3: 우수 달성** | 본인 업무 주제로 커스텀 워크플로우 설계 | 수강생 30% |

## 4. 도구 역할 매트릭스

| 도구 | 워크플로우 역할 | 핵심 기능 | 과정 내 비중 |
|------|---------------|-----------|-------------|
| **노트북LM** | ① 자료 수집·정리 | 소스 업로드, 교차 분석, 오디오 오버뷰, FAQ 생성 | 25% |

| **클로드** | ③ 구조화·시각화 | Artifacts(차트/그래프), Show Me(다이어그램), 보고서 작성 | 30% |
| **안티그래비티** | ④ 최종 결과물 제작 | 슬라이드 디자인, 인포그래픽, 보고서 레이아웃 | 20% |

### 도구 연동 워크플로우 (E2E)

```
[자료 수집]          [분석·리서치]        [구조화·시각화]      [최종 결과물]
  노트북LM     →      제미나이      →       클로드       →    안티그래비티
  ─────────         ──────────         ──────────         ──────────
  PDF/웹/영상       딥리서치 실행       데이터→차트 변환     슬라이드 완성
  소스 정리         트렌드 분석         보고서 구조 설계     인포그래픽
  핵심 요약         경쟁사 비교         다이어그램 생성      최종 디자인
  FAQ 추출          데이터 정제         텍스트 구조화       브랜딩 적용
```

#### C-3. 실습2: 클로드 스킬 라이브 — 맞춤 슬라이드 제작 (30분)

**컨셉**: "원하는 양식을 보여주면, AI가 그 양식대로 만들어 줍니다"

| 시간 | 단계 | 실습 내용 | "와!" 포인트 |
|------|------|----------|-------------|
| 5분 | **① 양식 분석** | 실제 기업 슬라이드/보고서 양식 1개를 클로드에 보여주며 "이 스타일대로 만들어주는 스킬을 만들겠습니다" 선언 | "내 양식을 학습시킨다고?" |
| 10분 | **② 스킬 제작 라이브** | 강사가 실시간으로 클로드 스킬(프로젝트) 제작: 양식 규칙 정의 → 프롬프트 설계 → 테스트 실행까지 전 과정 공개 | "이렇게 쉽게 만들 수 있어?" |
| 8분 | **③ 결과물 즉석 생성** | 방금 만든 스킬에 주제만 바꿔 입력 → 동일 양식의 슬라이드 3종 즉석 생성 (보고서/제안서/교육자료) | "주제만 바꾸면 양식은 자동!" |
| 7분 | **④ 수강생 체험** | 미리 준비된 스킬을 수강생에게 공유 → 본인 주제로 슬라이드 1장 직접 생성 | "내 자료가 프로 슬라이드로!" |

> **스킬 제작 시연 시나리오 (상세)**
> ```
> [라이브 시연 흐름]
> 1. 양식 제시: "이것은 S사 내부 보고서 양식입니다" (PPT 캡처 or PDF)
> 2. 클로드에 업로드: "이 양식의 레이아웃, 색상, 구조를 분석해줘"
> 3. 스킬 작성: 분석 결과를 바탕으로 프로젝트 지침 작성 (3분)
>    - 슬라이드 구조 규칙 (표지/목차/본문/요약)
>    - 톤앤매너 (색상, 폰트 느낌, 문장 스타일)
>    - 출력 형식 (Marp 마크다운 or HTML)
> 4. 테스트: "2026년 AI 시장 트렌드" 주제로 실행 → 결과물 즉시 공개
> 5. 변형: 같은 스킬로 "카페 창업 사업계획서" 즉석 생성 → 양식 일관성 확인
> ```

> **핵심**: "나도 우리 회사 양식으로 이걸 만들 수 있겠다!" → 본과정 수강 동기 극대화

#### C-4. 시연: "5분의 기적" + 결과물 갤러리 (20분)

**컨셉**: 4개 도구 조합 라이브 퍼포먼스 + 스킬로 만든 실제 결과물 갤러리 공개

```
[라이브 시나리오]
주제: "2026년 커피 시장 트렌드 보고서" (또는 청중 투표로 주제 선정)

Step 1 (2분): 노트북LM에 관련 기사 3개 업로드 → 핵심 인사이트 추출
Step 2 (3분): 제미나이 딥리서치로 시장 데이터 수집
Step 3 (3분): 클로드 스킬로 맞춤 양식 보고서 + 시각화 자동 생성
Step 4 (2분): 안티그래비티에서 최종 슬라이드 완성
```

**결과물 갤러리** (10분)

| 시간 | 내용 | 효과 |
|------|------|------|
| 5분 | **Before/After 갤러리**: 스킬 적용 전(일반 프롬프트) vs 적용 후(맞춤 스킬) 결과물 비교 5종 공개 — ① 기업 보고서 ② 마케팅 제안서 ③ 강의 슬라이드 ④ 소상공인 사업계획서 ⑤ 정책 보고서 | "양식만 바꾸면 뭐든 된다!" |
| 5분 | **"이 모든 결과물은 스킬 1개로 만들었습니다"** → 본과정 연결: "본과정에서는 여러분 회사/업무 양식에 맞는 스킬을 직접 만듭니다" + 얼리버드 안내 | 갈망 → CTA |

> **클라이맥스**: 5종 결과물을 화면에 나열하며 — "이 모든 것이 양식 1개 + 스킬 1개 + 주제만 바꾸기로 만들어졌습니다. 본과정에서는 여러분이 직접 이걸 합니다."
